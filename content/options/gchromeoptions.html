General

Automatically close panels after successfully posting. The content of the panel is deleted.

If you open a panel that is not entirely visible, this will scroll to the panel so it is visible.

Link Hijacking

Clicking on links makes a panel appear instead of going to the normal post form. Note that disabling these preferences will prevent you from accessing the core functionality of IGN++. The "Quick" Reply and Quote buttons, which open up IGN's default quick reply, are never hijacked.

The "Post New Topic" link on boards opens a topic panel.

The "Reply" links below posts and the main "Post Reply" link in threads open reply panels.

The "Quote" links below posts open reply panels with the post quoted inside. Links are also subject to Selection Quote rules.

On the Classic theme, "Private Message" links below posts open PM panels. On other themes, you can hover over the user's icon to reach this link instead. "Reply" links below PMs also open PM panels.

"Edit" links below posts open edit panels.

Redirection

This refers to bringing you to the page that IGN would normally bring you to after posting something. Only applies to actions performed by panels. It's not a good idea to enable redirection if you open multiple panels at once, since the content of any open panels is lost upon redirection.

After posting a topic, you are brought to the topic's page.

After posting a reply, you are brought to the page which that reply appears on.

After posting a reply without entering the thread, do the same thing reply redirection does.

After posting an edit, you are brought to the page which your edit appears on.

For each of the above, redirect to the target page's board instead. For example, if Topic redirection is on and you post a topic to the Vesti, then you will be redirected to the Vesti.

Integration

Integration refers to the placement of panels within the page as opposed to floating above it. If integration is disabled, panels will float above the page at the position determined by the Position preferences. Edit panels are always integrated so you can easily determine which post you are editing. PM panels opened on PM pages are always floating, as are Reply Without Entering panels.

Panels opened via Topic Hijacking are integrated.

Panels opened via Reply Hijacking are integrated.

Panels opened via PM Hijacking are integrated.

When you quote multiple people, the open panel moves so it's under the quoted post.

Autocensor

The autocensor will automatically censor bannable words in your text. Censoring occurs when the Post button in a panel is focused (in other words, when you tab over to it or click it).

The censor is not perfect. You are responsible for what you post.

Autocensor posts (Topics, Replies, Edits).

Try to censor words which are only bannable in certain contexts.

Use asterisks to censor the letters of bannable words. If disabled, IGN++ will instead replace bannable words with similar but non-bannable words.

Editors

Editors are the areas in panels which include the formatting buttons, faces, and WYSIWYG/Code inputs.

If this is enabled, editors will, by default, use the What-You-See-Is-What-You-Get view. In this view, you see your post as it will appear when you post it - images and formatting are visible, etc. If this is disabled, editors will instead use the Code view. In the Code view you can see the raw boardcode that will be posted. This allows finer control of the content of your post, but isn't as easy to use. Note that you can switch between the views freely - this preference just controls the default state.

If enabled, IGN++ will add its editor to the standard IGN post forms.

Pretext

Pretext refers to content which IGN++ can add to editors for you. You can use the [cursor] tag to indicate where the cursor should be placed after inserting text. You can use the [author] tag in the Before and After fields to refer to the author of the quoted post.

This text will be added to the start of every post.

This text will be added to the end of every post. IGN++ will also try to keep it at the end when you quote people.

This text is added before quoted text. [quote] tags are not included implicitly!

This text is added after quoted text. [/quote] tags are not included implicitly!

This text will be added at the end of your post when you edit it.

If you click "Quote" under a post when part of the post is selected, only the selected part is quoted. Note that you can make multiple selections by holding Control. IGN++ will quote each selection.

If this is enabled, you can only select inside the post you are trying to quote. If a selection is made outside of the post and you try to quote it, the full post is quoted as if Selection Quote were off.

If Strict Selection is off and you select outside of the post you're quoting, IGN++ will always try to use the correct username as the author. If this is disabled, IGN++ will just use the author of the post whose button you're clicking.

Autorefresh

Automatically refresh the content of the page. Refreshes occur after some interval, measured in milliseconds. Autorefresh can be temporarily disabled on a page by pressing 'A'. A refresh can be triggered by pressing 'R'. Most refreshes are indicated by a green border around the page.

Automatically refresh the topics on a board. If you hover your mouse over the topics area, it won't refresh.

Automatically refresh the replies in a topic. New replies will have a yellow gradient on the top.

After successfully posting, trigger a refresh so you can see your reply immediately.

If a post is edited, show the new post content. Edited posts have a blue gradient on the top.

If a poll is on the page, refresh its contents so you can see changes as they occur.

Refresh the unread Private Message count which appears over your icon towards the top of the page.

Refresh the contents of the Recent Posts area at the bottom of the page. Refreshes at the same interval that topics or replies refresh at (depending on what page you're on).

WULs

Stands for Watched User List. WULs are the de facto currency of the boards. Adding a user to your Watched User List (i.e. WULing them or giving them a WUL) is also used as a sign of respect or thanks.

Show links to WUL the author below posts in threads.

Automate the WULing process when you click on a WUL link. The link background will turn white, then yellow, then green. When it's green, WULing is complete. Don't leave the page before then. You can De-WUL somebody by Alt + clicking a WUL link. The link border is blue if you're WULing, purple if De-WULing. The border is dashed if this action will not change anything (i.e. you're WULing someone you've already WUL'd).

Ignore Users

IGN++ lets you ignore users so you don't see their replies and/or, optionally, their topics. You can add/remove users from your ignore list with the links below a user's post or by manually editing the list below.

Hide replies posted by users on your ignore list.

Hide topics posted by users on your ignore list.

This shows you which users are on your ignore list. It is a comma-separated list of usernames. Usernames are 3-20 characters long and can contain letters, numbers, underscores, periods, and dashes.

Show links to Ignore/Unignore and Show/Hide posts.

Position

The position of floating panels measured by the distance, in pixels, between the bottom left corner of the screen and the bottom left corner of the panel.

Within Topics

The position when on a topic page.

Elsewhere

The position when on all other pages.

Integrated Tools

Add controls for opening panels to the Boards drop-down menu in the navigation area of the page.

Add a Vesti Wiki search button to the board search area.

Redirect all board searches to a site-specific Google search.

Open all search results in a new tab.

Add a number of helpful links to the Boards drop-down menu.

Lets you browse the boards without a mouse. This feature is incomplete and broken in many ways. It will probably be removed in a later version.

Miscellaneous

If the reply, quote, or edit links are missing from a post, like in a locked thread, then add them back in with a lighter color and dashed border. The links may not work, but can be useful for quoting a post in a locked thread.

Show links to a user's Vesti Wiki page below their post.

Show ordinal numbers for each reply in a thread.

If you have arrived on a page whose URL contains a reply ID (.../r1234567/), scroll to the reply with the given ID if it exists.

If you click on a link to the thread you are currently in, and the link contains a reply ID, and the reply with the given ID exists on the page you're viewing, then scroll to the reply with the given ID instead of navigating to the URL.
